# Workflow — Competitor Analysis

## Purpose
Produce a fair, useful comparison between myFaro and a competitor or tool, framed around the operating-layer positioning.

## Owner agent
`agents/strategy-positioning-agent.md`

## Trigger
A prospect mentions a competitor, or a battlecard needs refreshing.

## Inputs
- Competitor / tool name (e.g. Abbove, Egor, Fundshop, Finary, Dory, Brio, Brokercloud, supplier portals).
- Optional: known prospect context.

## Steps
1. Pull existing framing from `memory/06_COMPETITIVE_INTELLIGENCE.md`.
2. Describe where each tool typically fits (CRM, planning, reporting, supplier portal, pension-specific).
3. Position myFaro as the horizontal operating layer across suppliers.
4. Identify fair differentiators and where the tools coexist.
5. Draft commercial talking points and objection responses.

## Output
- Comparison / battlecard summary.
- Objection responses (`templates/objection-handling.md`).

## Guardrails
- Use fair language; do not invent current competitor features.
- Frame myFaro as horizontal operating layer, not a like-for-like replacement.
- Acknowledge where systems can coexist.
